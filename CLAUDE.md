## Project: AgentFB

Full-stack admin panel for agentic Facebook auto-posting. Users configure brands (voice, post mix, schedule) and the system autonomously generates and publishes content.

## Tech stack

- Next.js 16 (App Router, TypeScript strict)
- Tailwind CSS v4 + shadcn/ui (light mode only)
- Supabase: Postgres, Auth, Storage, Edge Functions, pg_cron
- Claude Sonnet 4.5 (`claude-sonnet-4-5`) for content decisions
- Gemini 2.5 Flash Image (`gemini-2.5-flash-image`) for image generation
- Facebook Graph API v21.0
- Telegram Bot API for semi_auto approval pings
- Deployed on Vercel

## Commands

- `npm run dev` — start dev server on http://localhost:3000
- `npm run build` — production build
- `npm run lint` — eslint
- `npx tsc --noEmit` — typecheck without emit
- `npx shadcn@latest add <name> -y -o` — add a shadcn component

## Architecture

**Agent pipeline** — the orchestrator (`lib/agents/orchestrator.ts`) runs the full pipeline inside the Next.js app at `POST /api/trigger`. pg_cron hits that endpoint every 5 min with a shared `CRON_SECRET` header; the Supabase Edge Function `supabase/functions/run-agent` is a thin Deno wrapper that forwards to the same route (used when the cron extension can't reach the public URL directly).

1. Query `brands` where `mode ≠ 'paused'` and current time matches a `posting_time` in the brand's timezone (±10 min window).
2. For each due brand: load the user's per-service credentials from `api_keys` (base64-decoded), pick a `post_type` by weighted `post_mix`, fetch last 7 days of posts, call content-agent (Claude Sonnet 4.5) with brand config + history.
3. Claude returns strict JSON: `{ post_type, topic, hook, caption, cta, hashtags, image_prompt? }`. Validated with zod + banned-words filter.
4. If post needs an image: image-agent (Gemini 2.5 Flash Image) generates → uploaded to the `post-images` Supabase Storage bucket → public URL.
5. `mode='semi_auto'` → insert with `status='pending_approval'` + Telegram ping. `mode='auto'` → insert with `status='approved'`, publisher-agent posts to FB Graph v21.0, then `status='posted'`.
6. Every step logs to `logs` via `lib/agents/logger.ts`.

Manual trigger: `RunNowButton` on `/` (all due brands) and on each brand detail page (`force=true` for that brand).

**Database tables**:
- `supabase/migrations/001_initial_schema.sql` — `brands`, `posts`, `api_keys`, `logs` + RLS policies.
- `supabase/migrations/002_agent_pipeline.sql` — `post-images` storage bucket + pg_cron schedule.

**Secrets flow**: user enters API keys in the settings page → stored in `api_keys` table (base64 for MVP, Supabase Vault later) → edge functions fetch from DB at runtime. Never hardcode.

## Folder conventions

- `app/(dashboard)/` — route group for authenticated admin pages, shares sidebar layout
- `app/api/` — Next.js route handlers for the admin panel
- `supabase/functions/` — Deno-based edge functions (different runtime from the Next.js app)
- `lib/supabase/client.ts` — browser client; `lib/supabase/server.ts` — server client with cookie-based session
- `lib/agents/` — agent logic (content, image, publisher)
- `lib/prompts/` — system prompt templates
- `components/ui/` — shadcn primitives (don't edit directly; use `shadcn add` to update)

## Critical rules

- Server Components by default. Use `"use client"` only for forms, interactivity, or `usePathname`/`useState` hooks.
- All API routes validate input with **zod**.
- All forms use shadcn/ui components (`Form`, `Input`, `Select`, etc.).
- Wrap risky operations in try/catch, log to `logs` table, surface to user via `sonner` toast.
- Never expose `SUPABASE_SERVICE_ROLE_KEY` to the client — use it only in server routes and edge functions.
- Destructive UI actions (delete brand, reject post) require a confirmation `Dialog`.

## Content voice

Content generated by the Claude agent must be **Taglish** (natural Filipino-English mix) for the PH market. Follow Hormozi / Halbert / Dan Kennedy copywriting: Hook → Body → CTA. Respect `brand.banned_words`. Avoid format/topic repetition vs. the last 7 days (pass history into prompt).
